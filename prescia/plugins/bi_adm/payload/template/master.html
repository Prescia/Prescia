Ajuda ?
- Clique no ícone de reset para limpar as caches e recarregar os XML

CODE - O código do cliente na programação

DEBUG - Informa se o site está rodando em modo debug. Se sim, ele carrega os dados do XML e mostra a barra de debug (o "^" no rodapé), o que deixa o site lento

CONFIG - Informa se existem campos importantes faltando na configuração (no administrativo, em opções)

QUOTA - Informa se o cliente está próximo ou excedeu a quota de disco

ERROS - Informa se ocorreram erros (inclui apenas avisos) no site durante o dia atual

404 - Informa se estão visitando páginas inexistentes (404) no site, durante o dia atual. MUITO pode ser tentativa de hack ou site com defeito

CRON - Informa se as tarefas agendadas automáticas do sistema (CRON) já rodaram na HORA atual, é comum haver um atraso se o site não é visitado com frequência. Se continuar o alerta mesmo depois do site visitado, então algo está errado

HITS - Informa quantos hits o site teve hoje, seguido do total nos últimos 7 dias. Se as visitas hoje forem consideradas muito acima da média, alerta para a possibilidade de ataque ou slashdot

BOT - Informa a visitação de bots, e se ela é normal ou acima do esperado. Visitação excessiva pode indicar ataque

TIME - Informa qual foi a página que mais demorou a ser servida HOJE.

BACKUP - Informa o status do backup automático de banco de dados, e o dia programado para ele ocorrer (sql gravado na pasta _backup dentro da nekoi do servidor).



Note que "ERROS" e "404" são relativos apenas a dados do dia atual (resetado a meia noite), e CRON relativo apenas a hora atual (pois o CRON ocorre a cada hora). Os avisos sobre estatísticas de visitação (HITS, BOTS e TIME) são relativos também ao dia atual.

Você pode clicar no ícone de erro de um site para ver o log de erros do mesmo.

O log de erros e ações é mantido pelo sistema por 3 meses (exibido no adminstrativo do site os últimos 3 dias, aqui apenas do dia atual)

O log de 404 é mantido apenas para o dia atual e zerado a meia noite.
